# Phrase Dictionary Sample TSV

This folder contains `phrase_dictionary_sample_100.tsv`, a 100-row sample dictionary generated from the phrase research files.

## Source Files

The TSV is generated by `phrase-extract/scripts/build_phrase_score_sample.py` from these files:

- `phrase-extract/research/dela-fr-public-u8.phrases.tsv`
- `phrase-extract/research/idioms-runtime-dictionary.tsv`
- `phrase-extract/research/FleLex_CRF.csv`

## Merge Rules

Rows from DELA and the idiom dictionary are merged by normalized phrase text:

- lowercased text
- normalized apostrophes
- collapsed whitespace
- DELA rows use `base` as the main phrase and keep `surface` values in `surface_forms`
- idiom rows use `entry` as the main phrase

If the same phrase appears in both DELA and the idiom dictionary, the output row has `sources=dela|idiom`.

## FleLex Mapping

FleLex is used to estimate lexical rarity and lexical difficulty.

The script first tries an exact normalized phrase match in FleLex. If that fails, it tokenizes the phrase and scores known tokens. If no tokens are found in FleLex, the row gets a fallback rarity value.

The `flelex_match` column shows which path was used:

- `exact`
- `tokens`
- `none`

## Scores

The old score is now called `confidence_score`. It means candidate reliability, not learner difficulty:

```text
confidence_score =
0.55 * lexical_score
+ 0.25 * confidence_idiom_score
+ 0.10 * type_score
+ 0.10 * length_score
```

The learner difficulty score is `difficulty_score`:

```text
difficulty_score =
0.45 * rarity_score
+ 0.25 * cefr_proxy_score
+ 0.15 * semantic_opacity
+ 0.06 * register_score
+ 0.05 * construction_fixedness
+ 0.02 * structural_complexity
+ 0.02 * idiom_strength_score
```

## Difficulty Components

`frequency_total`: The raw FleLex frequency total used for exact phrase matches. For token-only matches, the script uses the lowest known token frequency as a conservative proxy.

`rarity_score`: Higher means less common. It comes from FleLex when available and is now the largest difficulty component. Common calibrated learner expressions such as `au lit` are prevented from being over-penalized by sparse corpus counts.

`cefr_proxy_score`: Estimated learning stage. It uses FleLex CEFR-level frequency when available, frequency bands as a proxy, and explicit calibration anchors for known expressions such as `bien sûr`, `au lit`, `appareil photo`, and `a priori`.

`semantic_opacity`: Higher means the phrase is harder to guess from literal word meanings. It now distinguishes transparent compositional expressions such as `appareil photo` and `au lit`, fixed collocations such as `de peur de`, and opaque learned units such as `a priori` and `bouton-d'or`.

`construction_fixedness`: Higher means the phrase behaves more like a fixed construction that must be learned as a unit. Transparent nominal and prepositional phrases receive lower values than opaque idioms or formal fixed expressions.

`structural_complexity`: Based on phrase shape:

- 2 words or fewer: `0.2`
- 3 words: `0.4`
- 4+ words: `0.6`
- forms containing `que`, `où`, or `à ce que`: `0.8`

`register_score`: Estimated from phrase text:

- everyday/spoken: `0.1`
- neutral: `0.3`
- formal/written: `0.7`
- legal/technical/literary: `0.9`

`idiom_strength_score`: Based on idiom dictionary strength:

- none/free lexeme: `0.0`
- weak idiom: `0.3`
- semi-idiom: `0.6`
- strong idiom: `0.9`

This score is intentionally low-weighted in `difficulty_score`, because idiomaticity is not the same as learner difficulty. High-frequency idioms can be easy; rare or academic fixed expressions can be difficult.

`length_score`: Kept as a diagnostic column for the legacy `confidence_score`, but it no longer contributes to `difficulty_score`.

## Calibration Check

The generator validates a small learner ranking before writing the TSV:

- `bien sûr` < `appareil photo`
- `au lit` < `a priori`
- `appareil photo` < `a priori`
- `bien sûr` < `bouton-d'or`

## Important Column

`scoring_form` is the phrase form used for structural complexity. This matters because DELA sometimes stores a shorter `base` than the real surface phrase. For example, `de manière à ce que` has base `de manière`, so the output phrase is `de manière` but `scoring_form` keeps the longer surface form.
